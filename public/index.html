Vibe Coding at Trivelta.

Ship internal apps in under 10 minutes. Pick a route below — clicks or commands — and you'll have a live URL by the end of this page.

Two routes, same destination

Route A

Click through the dashboards

For when you'd rather see the buttons. Five steps, all browser, no terminal required.

1

Get access

Get invited to the Trivelta Vibe Coders GitHub org. Join the Trivelta Vibe Coders Railway workspace via invite link.

2

Create your project

Open the Trivelta Vibe Coders org on GitHub. Click Use this template on the app-template repo. Name it whatever you want — it's yours to vibe with.

3

Deploy to Railway

Open the Railway dashboard at railway.app/dashboard. Click New Project → Deploy from GitHub Repo. Select your new repo (it must live in the Trivelta Vibe Coders org). Railway auto-detects your stack and deploys instantly.

4

Get your public URL

In Railway, open your service's Settings → Networking. Click Generate Domain for a live up.railway.app URL.

5

Start vibing

  • Push to main — Railway auto-redeploys on every push.
  • Use Claude Code or any AI tool to iterate fast.
  • Add env vars in Railway: Service → Variables.
  • Need a database? Click + New → Database (Postgres, Redis, etc.).
Route B

One command, five steps

For when you'd rather type. Same outcome — template, repo, Railway project, domain, env scaffolding — wrapped into a single script. New to the terminal? You only need to copy, paste, and answer a couple of prompts.

Available now

$ git clone https://github.com/Trivelta-Vibe-Coders/trivelta-vibe-coders-guides
$ bash trivelta-vibe-coders-guides/scripts/vibe-bootstrap.sh my-app

The script walks you through GitHub auth, Railway auth, and picks sensible defaults. Replace my-app with your project name.

Coming soon — one-liner

$ curl -fsSL https://vibe-coders-guides.up.railway.app/install.sh | bash -s my-app

No clone, no repo on disk. Curls the bootstrap script and runs it in place.

Coming soon — npm

$ npx @trivelta/vibe init my-app

The eventual UX once we publish to npm. Same flow, nicer entry point.

·

What the script does

  • Forks the app-template into the Trivelta Vibe Coders org under your name.
  • Creates a Railway project, links the repo, and triggers the first deploy.
  • Generates a public up.railway.app domain.
  • Drops a starter .env.example and a CLAUDE.md so Claude Code or Cursor know the lay of the land.
  • Prints the live URL when it's done.

When it breaks

Build failed

Open the failed deployment in Railway and read the build logs. Almost always a missing dependency or a typo in package.json.

App crashes on boot

Make sure your app listens on process.env.PORT. Railway assigns the port — hardcoding 3000 won't work in production.

Repo missing in Railway

It must live in the Trivelta Vibe Coders org, not your personal GitHub account. Move it, or recreate from the template inside the org.
