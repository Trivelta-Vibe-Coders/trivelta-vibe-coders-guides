Vibe Coding at Trivelta.

Ship internal apps in under 10 minutes. Pick a route below — clicks or commands — and you'll have a live URL by the end of this page.

Two routes, same destination

Route A

Click through the dashboards

For when you'd rather see the buttons. Five steps, all browser, no terminal required.

1

Get access

Ask Connor (or whoever pointed you here) for a Trivelta Vibe Coders GitHub org invite. Then join the Railway workspace with this invite link.

2

Create your project

Open app-template and click Use this template (direct link). Name it whatever you want — keep it inside the Trivelta Vibe Coders org so Railway can see it.

3

Deploy to Railway

Open railway.app/dashboard and switch to the Trivelta Vibe Coders workspace. Click New Project → Deploy from GitHub Repo and pick your new repo. The Railway GitHub App is already installed for the org, so every repo here appears automatically. Railway auto-detects your stack and deploys.

4

Get your public URL

In Railway, open your service's Settings → Networking. Click Generate Domain for a live up.railway.app URL.

5

Start vibing

  • Push to main — Railway auto-redeploys on every push.
  • Iterate with Claude Code, Cursor, or whatever AI tool you like.
  • Env vars: service Variables tab.
  • Need a database? + New → Database (Postgres, Redis, MySQL, Mongo).
Route B

One command, five steps

For when you'd rather type. Same outcome — template, repo, Railway project, domain, env scaffolding — wrapped into a single script. New to the terminal? You only need to copy, paste, and answer a couple of prompts.

Prereqs

$ brew install gh railway jq
$ gh auth login                       # scopes: repo, read:org, workflow
$ railway login

One-time setup. Skip what you already have.

Available now

$ git clone https://github.com/Trivelta-Vibe-Coders/trivelta-vibe-coders-guides
$ bash trivelta-vibe-coders-guides/scripts/vibe-bootstrap.sh my-app

Replace my-app with your project name (lowercase, hyphenated). Add --db postgres for a managed database, --public to make the repo public.

Coming soon — one-liner

$ curl -fsSL https://vibe-coders-guides-production.up.railway.app/install.sh | bash -s my-app

No clone needed. Curls the bootstrap script and runs it in place.

Coming soon — npm

$ npx @trivelta/vibe init my-app

Once we publish to npm. Same flow, nicer entry point.

·

What the script does

  • Creates a new repo from app-template in the Trivelta Vibe Coders org (private by default; --public flips it).
  • Clones it locally next to your other projects.
  • Spins up a Railway project in the Trivelta Vibe Coders workspace and links the repo as the service source.
  • Optionally provisions a managed database (--db postgres|redis|mysql|mongo).
  • Generates a public *.up.railway.app domain.
  • Prints repo URL, Railway URL, and live URL — usually in under a minute.
!

After bootstrap — confirm push-to-deploy

Open your service in Railway → Settings → Source. The repo should already be connected. If you see No source, click Connect Repo and pick your new repo — without this, pushes to main won't trigger a redeploy. One-time check.

When it breaks

Build failed

Open the failed deployment in Railway and read the build logs. Almost always a missing dependency or a typo in package.json.

App crashes on boot

Make sure your app listens on process.env.PORT. Railway assigns the port — hardcoding 3000 won't work in production.

Repo missing in Railway

It must live in the Trivelta Vibe Coders org, not your personal GitHub account. Move it, or recreate from the template inside the org.
